1.2 初识输入输出

iostream库 包含 istream 和 ostream两个基础类型，分别表示输入流和输出流。
流：随着时间的推移，字符式顺序生成或消耗的。

标准输入输出对象

标准库定义了4个I/O对象。
cin 标准输入
cout 标准输出
cerr 标准错误 输出警告和错误消息
clog 输出程序运行时的一般性消息

向流写入数据

输出运算符(<<)能接受两个运算对象，左侧是 一个ostream对象，右侧对象是 要打印的值。

从流读取数据

输入运算符(>>) 接受一个istream作为左侧运算对象，接受一个对象作为其右侧运算的对象。从给定的istream读入数据，并存入给定对象中。

使用标准库中的名字

  • 前缀std::指出名字cin、cout和endl是定义在名为std的命名空间（namespace）中的。
    标准库定义的所有名字都在命名空间std中。

优点：命名空间可以帮助我们避免不经意的名字定义冲突，以及使用库中相同名字导致的冲突。
缺点：使用标准库中的一个名字时，必须显式说明标准想使用的命名空间的名字。 例如：std::cout，通过使用作用域运算符(::) 来指出定义在命名空间std中的名字cout。

  • endl 是被称为操纵符（manipylator)的特殊值。
    写入endl的效果是结束当前行，并将与设备关联的缓冲区（buffer)中的内容刷到设备中。缓冲刷新操作可以保证到目前为止程序所产生的所有输出都真正写入输出流中。

1.3 注释简介

C++中有两种注释：单行注释和界定符对注释。

  1. 单行注释以双斜线（//）开始，以换行符结束。
    当前行双斜线右侧的所有内容都会被编译器忽略，可以包含任何文本，包括额外的双斜线。
  2. 使用两个界定符（ / 和 / ） 。以 / 开始，以 / 结束，可以包含除 * / 以外的任何内容。

注释界定符不能嵌套

1.5 类简介

数据结构（data structure)
C++中，通过哟定义一个类（class）来定义自己的数据结构。
一个类定义一个类型，以及与其相关联的一组操作。
C++能定义使用上像内置类型一样自然的类类型（class type）。

为了使用类，需要关注的三件事：

  1. 类名
  2. 定义的位置
  3. 支持的操作

使用头文件来访问为自己应用程序所定义的类。
习惯上，头文件根据其中定义的类的名字来命名。
头文件的后缀一般使用 .h ，也有 .H 、.hhp 或 .hxx 。 标准库头文件通常不带后缀。

使用文件重定向

大多数操作系统支持文件重定向——一种允许我们将标准输入和标准输入与命名文件关联起来的机制：
$ additemsoutfile
假定：

  • $ 操作系统提示符
  • 加法程序已经编译名为additems.exe的可执行文件（UNIX中是additem)

上述命令会从一个名为infile的文件读取销售记录，并将结果输入到outfile的文件中，两个文件都应位于当前目录中。

1.5.2 初识成员函数

成员函数（member function)是定义为类的一部分函数，有时也被称为方法（method)。

点运算符（ . ）只能用于类类型的对象。其左侧运算对象必须是一个类类型的对象，右侧运算对象必须是该类型的一个成员名，表达”名为 xxx 的对象的 xxx 成员“。

用调用运算符（ () ）来调用一个函数，调用运算符包括一对圆括号，用来放置实参（argument）列表（可为空）。
若成员函数不接受参数，结果为调用成员函数本身。

1.6 书店程序

术语表

  • 缓冲区（buffer)一个存储区域，用于保存数据。IO设施通常将输入（输出）数据保存在一个缓冲区中，读写缓冲区的动作与程序中的动作是无关的。我们可以显式刷新输出缓冲，以便强制将缓冲区中的数据写入输出设备。默认情况下，读cin会刷新cout;程序非正常终止也会刷新cout.

  • 内置类型（built-in type）由语言定义的类型，如int。

  • Cerr 一个ostream对象，关联到标准错误，通常写入到与标准输出相同的设备。
    默认情况下，写到cerr的数据是不缓冲的。cerr 通常用于输出错误信息或其他不属于程序正常逻辑的输出内容。

  • clog 一个ostream对象，关联到数据标准错误。
    默认情况下，写到clog的数据是被缓冲的。clog通常用于写报告程序的执行信息，存入一个日志文件中。

数据结构（data structure)数据及其上所允许的操作的一种逻辑组合。

文件结束符（end-of-file）系统特定的标识，指出文件中无更多数据了。
